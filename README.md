# Engineering Notes

Daily engineering notes covering systems design, debugging insights, architecture decisions, and lessons learned while building software.

## Structure

```
interviews/       # One note per interview — copy TEMPLATE.md for new ones
systems-design/   # Design sketches, tradeoffs, practice problems
debugging/        # Hard bugs, root causes, war stories
architecture/     # Decisions, ADRs, lessons learned
```

**Naming:** Use `YYYY-MM` (or `YYYY-MM-DD`) at the start of filenames so everything sorts chronologically, e.g. `2025-03-acme-backend.md` in `interviews/`, or `2025-03-17-cache-invalidation.md` in other folders.

## What's in here

- **Interview notes** — Companies, roles, what I was asked, and what I’d do differently next time  
- **Systems design** — Sketches, tradeoffs, and post-mortems  
- **Debugging & architecture** — Hard bugs, root causes, and design decisions worth remembering  

## Interview tracking

While job hunting, I use this repo to:

- Log each interview (company, role, round, date)
- Capture questions (coding, system design, behavioral)
- Reflect on what went well and what I’d improve
- Build a reusable reference for prep and future interviews
